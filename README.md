# huashu-research · 结构化网络调研

> 五阶段门禁调研流程。每阶段不过即停。成果增量保存到文件，不因会话截断丢失。

当用户说"调研"、"搜索资料"、"帮我查一下"时触发。不是"搜一下看看"——是结构化研究流程：搜索→筛选→深度阅读→对比分析→报告产出。每个阶段有明确的输入、输出和门禁条件。

## 快速开始

```
> 帮我安装这个 skill：https://github.com/wampeeHuang/huashu-research
```

触发词：调研、搜索资料、帮我查一下、研究一下

## 设计哲学

### 增量落盘，不靠会话记忆
每个调研阶段的结果写文件，不是靠 AI 上下文硬记。会话断了→文件在→调研继续。这是工程规则，不是好习惯——AI 的上下文窗口是调研的最大瓶颈。

### 阶段门禁：不合格不进下一段
每阶段有明确的通过标准。搜索不够深→不进入筛选。筛选不够准→不进入深度阅读。门禁是硬的——不是建议，是代码逻辑。门禁比流程更重要的原因是：跳过的步骤在报告里一定看得出来。

### 工具优先于直觉
结构化的搜索步骤（opencli、WebSearch、WebFetch）优先于 AI 的"我觉得"。调研不是 AI 在回忆训练数据——是 AI 在操作工具获取当前世界的信息。

## 诚实边界

- **中文搜索为主。** 英文源需额外配置搜索源
- **不替代学术研究。** 深度和严谨性不足以支撑学术论文级别
- **实时性取决于搜索工具。** 受限于可用的搜索 API 和网络条件
- **不适合"一句话问题"快速查询**——那是 smart-search 的场景
